Webhooks

Receive real-time notifications when payment events occur. HMAC-signed payloads delivered to your endpoint with automatic retry on failure.

Event Types

EventTriggerDescription
payment.createdPayment initializedFires when a new payment is created via API or dashboard
payment.detectedTransaction seen on-chainFires when an incoming transaction matching the payment is detected (unconfirmed)
payment.confirmingAwaiting confirmationsFires when the transaction is in the mempool/has partial confirmations
payment.confirmedFully confirmedPayment has reached the required number of blockchain confirmations
payment.failedPayment failedTransaction reverted, insufficient amount, or expired without payment
payment.expiredPayment window closedNo payment received within the configured time window
withdrawal.completedFunds sentAutomated or manual withdrawal has been broadcast and confirmed
dispute.openedDispute raisedA customer has opened a dispute against this payment
dispute.resolvedDispute closedDispute resolved — refunded or dismissed

Webhook Security

Every webhook payload is signed with HMAC-SHA256 using your webhook secret. Always verify the signature before processing to prevent tampering.

  • ✓Signature header: x-xcg-signature
  • ✓Algorithm: HMAC-SHA256(raw body, webhook secret)
  • ✓Retry policy: 3 attempts with exponential backoff (5s, 30s, 5min)
  • ✓Timeout: Your endpoint must respond within 10 seconds
  • ✓Success: Return HTTP 200 to acknowledge receipt

Verify Signature (Node.js)
const crypto = require('crypto');

function verifyWebhook(body, sig, secret) {
  const hash = crypto
    .createHmac('sha256', secret)
    .update(JSON.stringify(body))
    .digest('hex');
  return hash === sig;
}
